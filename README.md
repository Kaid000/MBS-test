Проект реализует алгоритм поиска оптимального маршрута робота по складу с минимальными затратами.
Поддерживаются порталы, взвешенные клетки и стены. Используется алгоритм Дейкстры.

Структура проекта
Domain – модели: Cell, Grid, Position, CellType.
Application – бизнес-логика: DijkstraPathFinder, стратегии стоимости, провайдер соседей.
Infrastructure – загрузка данных: TextGridLoader.
Service – консольное приложение для запуска.

Требования
.NET 10
NUnit и Shouldly для юнит-тестов

Сборка и запуск
Через Visual Studio
Открой решение (.sln) в Visual Studio.
Установи проекты как Startup Project, выбери проект Service.
Нажми Ctrl + F5 или F5 для запуска.
Ввод данных:
Через stdin – просто вставляешь карту склада в консоль и завершаешь ввод (Ctrl+Z + Enter в Windows, Ctrl+D в Linux).

Через файл – запускаешь с аргументом:

--input path/to/file.txt
Через командную строку

Перейди в каталог проекта:

cd path/to/Service

Сборка проекта:

dotnet build

Запуск с консольным вводом:

dotnet run --

Запуск с файлом:

dotnet run -- --input path/to/file.txt
Формат входных данных
Первая строка: N M – размеры карты.
Далее N строк по M символов:
S – старт
E – финиш
. – обычная клетка (стоимость 1)
# – стена
1..9 – клетка со стоимостью входа от 1 до 9
A..Z – портал

Пример:

5 7
S..2..A
.#.#...
..#..#.
A..3..E
.......
Вывод
Одно число – минимальная стоимость пути из S в E.
Если путь невозможен – -1.

Пример вывода для предыдущей карты:

6
Юнит-тесты
Используется NUnit и Shouldly.
Тесты находятся в проекте Tests.
Запуск тестов через Visual Studio: Test Explorer → Run All.

Запуск через CLI:

dotnet test
Особенности реализации
Алгоритм Дейкстры с поддержкой порталов.
Стратегия стоимости для разных типов клеток.
Провайдер соседей учитывает стены и порталы.
Гибкая архитектура: DI через Microsoft.Extensions.DependencyInjection.
